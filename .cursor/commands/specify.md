# /specify コマンド

## 目的
Phase 1: Specify — ユーザー価値に集中した仕様定義を行います。

## 実行内容
1. ユーザーからの要求を分析し、ユーザーストーリーと受け入れ基準を明文化
2. 機能要件・非機能要件を整理
3. 曖昧な点には `[NEEDS CLARIFICATION]` マーカーを付与
4. **Best-of-N**: 可能であれば2案以上を提示し、推奨案を明示

## 成果物
`specs/<ticket-id>-<slug>/spec.md` を以下の構造で作成してください：

```markdown
# [チケットID] [機能名]

## ゴール
- 解決したいユーザー課題
- 成功指標や期待するビジネスインパクト

## コンテキスト
- 現在の痛みや目的
- 対象ユーザー/役割
- 想定される利用シーンやユーザージャーニー

## ユーザーストーリー
- As a [role], When [situation], I want [goal] so that [outcome].

## 機能要件
- [ ] 要件 1
- [ ] 要件 2

## 非機能要件
- 性能 / 信頼性 / セキュリティ / 運用の期待値

## 受け入れ基準（テスト観点）
- [ ] 観点 1
- [ ] 観点 2

## 曖昧さ・前提
- [NEEDS CLARIFICATION] 追加調査やユーザー確認が必要な点
```

## 注意事項
- Next.js 15 / React 19 のプロジェクトルールを考慮
- `.cursor/rules/` の内容（アーキテクチャ、コンポーネント設計など）を参照
- 技術的な実装詳細はまだ含めない（Phase 2: Plan で扱う）

## 次のステップ
仕様が確定したら、`/plan` コマンドで技術計画に進んでください。
