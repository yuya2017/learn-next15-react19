# Knowledge Maintenance Checklist

タスク完了時に `/knowledge` フェーズで確認するチェックリストです。

## 更新フロー

1. PR 差分にドキュメント更新が含まれているか確認
2. 新しい知見・ハマりどころを `docs/knowledge/<yyyy-mm-dd>-<topic>.md` として追加
3. `.cursor/rules` から参照される重要ルールに影響がある場合は該当ファイルも更新

## 記載フォーマット

```md
# 2025-01-01 tailwind-dark-mode-toggle

## 背景
- 課題や期待値を 2-3 行で記述

## 対応
- 実施したステップを箇条書きで整理

## 再発防止 / Tips
- 次回同様のタスクを行う際に注意すべき点
```

## レビュー観点

- 情報は具体的で再使用可能か？
- ソースコードや PR へのリンクが含まれているか？
- ドキュメントと実装の差分が無いか？
